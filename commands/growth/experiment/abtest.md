# Experimentation - A/B 테스트 설계

> **Shaul Corporation 원칙 적용**
>
> 데이터와 근거 기반 의사결정을 위해 체계적인 실험을 설계합니다.

A/B 테스트를 설계하고 분석합니다.

## 실행 단계

### 1단계: 실험 목표 정의

다음 정보를 수집하세요:

1. **실험명**: 실험 이름
2. **가설**: 테스트할 가설
3. **주요 지표**: 측정할 KPI

인자로 제공된 경우: $ARGUMENTS

### 2단계: 실험 설계

```markdown
## 실험 설계서

### 기본 정보
- **실험명**: [이름]
- **실험 ID**: EXP-[YYYYMMDD]-[번호]
- **담당자**: [이름]
- **기간**: [시작일] ~ [종료일]

### 가설
- **귀무가설 (H0)**: [변경이 효과 없음]
- **대립가설 (H1)**: [변경이 효과 있음]

### 변수
- **독립변수**: [변경하는 요소]
- **종속변수**: [측정하는 지표]
- **통제변수**: [고정하는 요소]

### 변형 (Variants)
| 변형 | 설명 | 트래픽 비율 |
|------|------|------------|
| Control (A) | 현재 버전 | 50% |
| Treatment (B) | 변경 버전 | 50% |
```

### 3단계: 샘플 크기 계산

```markdown
## 샘플 크기 계산

### 입력값
- 현재 전환율: [X]%
- 최소 감지 효과 (MDE): [Y]%
- 통계적 유의수준 (α): 0.05
- 검정력 (1-β): 0.80

### 계산 결과
- 변형당 필요 샘플: [N]명
- 총 필요 샘플: [2N]명
- 예상 실험 기간: [D]일

### 공식
n = 2 × ((Zα/2 + Zβ)² × p × (1-p)) / (MDE)²
```

### 4단계: 실험 실행 체크리스트

```markdown
## 실행 체크리스트

### 실험 전
- [ ] 가설 문서화
- [ ] 샘플 크기 계산
- [ ] 트래픽 분배 설정
- [ ] 추적 코드 구현
- [ ] QA 테스트 완료

### 실험 중
- [ ] 데이터 수집 확인
- [ ] SRM(Sample Ratio Mismatch) 체크
- [ ] 이상치 모니터링
- [ ] 조기 종료 기준 확인

### 실험 후
- [ ] 통계 분석
- [ ] 결과 문서화
- [ ] 의사결정
- [ ] 학습 공유
```

### 5단계: 결과 분석 템플릿

```markdown
## 실험 결과

### 요약
| 지표 | Control | Treatment | 차이 | p-value | 유의성 |
|------|---------|-----------|------|---------|--------|
| 전환율 | - | - | - | - | Yes/No |
| 보조지표1 | - | - | - | - | - |

### 결론
- **결과**: [승리/패배/무승부]
- **신뢰구간**: [X% ~ Y%]
- **권장사항**: [적용/재실험/기각]

### 학습
- [인사이트 1]
- [인사이트 2]
```

### 6단계: 산출물 생성

`docs/growth/experiments/[실험ID]-result.md` 생성

## 주의사항

- 동시에 여러 실험 시 상호작용 효과 주의
- 최소 1-2 비즈니스 사이클 실행
- Novelty Effect 고려
- Peeking Problem 방지 (조기 결론 금지)
