# Tasks - 팀별 태스크 관리

> **팀별 태스크 완료/미완료 현황을 관리합니다.**
>
> - Domain > Feature 계층 구조로 관리
> - Markdown 문서 형식
> - ./docs/tasks 폴더에서 관리

## 사용법

```
/tasks:{team}
```

예시:
- `/tasks:backend` - Backend 팀 태스크 확인
- `/tasks:frontend` - Frontend 팀 태스크 확인
- `/tasks:devops` - DevOps 팀 태스크 확인

## 인자

- `$ARGUMENTS`: 팀 이름 (예: backend, frontend, devops, qa, etc.)

## 실행 단계

### 1단계: 팀 이름 확인

인자로 팀 이름이 제공된 경우: **$ARGUMENTS**

팀 이름이 제공되지 않은 경우 AskUserQuestion 도구를 사용하여 다음 팀 중 선택하도록 요청하세요:
- backend
- frontend
- devops
- qa
- pm
- design
- data
- security
- infra
- sre

### 2단계: 태스크 문서 확인

`./docs/tasks/{team}.md` 파일을 읽어 해당 팀의 태스크 현황을 확인하세요.

파일이 존재하지 않는 경우, 새 태스크 문서를 생성할지 사용자에게 AskUserQuestion으로 질문하세요.

### 3단계: 태스크 현황 요약

태스크 문서를 분석하여 다음 정보를 요약해서 출력하세요:

```
## {팀명} 팀 태스크 현황

### 요약
- 전체: {total}개
- 완료: {completed}개 ({percentage}%)
- 미완료: {pending}개

### Domain별 현황

#### {Domain 1}
| Feature | 상태 | 담당자 | 마감일 |
|---------|------|--------|--------|
| feature1 | ✅ 완료 | @name | 2025-01-15 |
| feature2 | ⏳ 진행중 | @name | 2025-01-20 |
| feature3 | ❌ 미완료 | @name | 2025-01-25 |

#### {Domain 2}
...
```

### 4단계: 태스크 관리 옵션

사용자에게 다음 작업 중 하나를 선택하도록 AskUserQuestion으로 질문하세요:

1. **태스크 추가**: 새로운 태스크 추가
2. **태스크 완료 처리**: 기존 태스크를 완료로 변경
3. **태스크 삭제**: 태스크 제거
4. **종료**: 현황 확인만 하고 종료

#### 태스크 추가 시

다음 정보를 수집하세요:
- Domain (기존 domain 중 선택 또는 새로 생성)
- Feature 이름
- 담당자 (optional)
- 마감일 (optional)
- 설명 (optional)

태스크 문서를 업데이트하세요.

#### 태스크 완료 처리 시

미완료 태스크 목록을 보여주고, 완료할 태스크를 선택하도록 하세요.
선택된 태스크의 상태를 `✅ 완료`로 변경하세요.

#### 태스크 삭제 시

태스크 목록을 보여주고, 삭제할 태스크를 선택하도록 하세요.
선택된 태스크를 문서에서 제거하세요.

## 태스크 문서 구조

`./docs/tasks/{team}.md` 파일 구조:

```markdown
# {Team} 태스크 관리

## {Domain 1}

### {Feature 1}
- **상태**: ✅ 완료 | ⏳ 진행중 | ❌ 미완료
- **담당자**: @name
- **마감일**: YYYY-MM-DD
- **설명**: 태스크에 대한 설명

### {Feature 2}
...

## {Domain 2}

### {Feature 3}
...
```

## 상태 표시

| 상태 | 표시 | 설명 |
|------|------|------|
| 완료 | ✅ 완료 | 작업이 완료됨 |
| 진행중 | ⏳ 진행중 | 현재 작업 중 |
| 미완료 | ❌ 미완료 | 아직 시작하지 않음 |
| 보류 | ⏸️ 보류 | 임시 중단 |
| 취소 | 🚫 취소 | 작업 취소됨 |

## 주의사항

- 모든 문서는 UTF-8 인코딩을 사용합니다.
- 문서는 한글(Korean)로 작성합니다.
- Domain과 Feature 이름은 영문 소문자와 하이픈(-)을 사용합니다.
- 태스크 문서가 없는 경우 자동으로 템플릿을 생성할 수 있습니다.
