# Team Discussion Orchestrator

## 역할

여러 팀이 하나의 의제에 대해 협업하여 토론하고 합의에 도달하는 오케스트레이터입니다.

## 워크플로우

```
┌─────────────────────────────────────────────────────────────────────┐
│                    /team-discussion 실행                              │
└────────────────────────────┬────────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│  1단계: 의제 설정                                                     │
│  - 토론 주제, 참여 팀, 결정 유형                                       │
└────────────────────────────┬────────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│  2단계: 1차 의견 수집 (병렬)                                           │
│  ┌─────┐  ┌─────┐  ┌─────┐  ┌─────┐                                │
│  │팀 A │  │팀 B │  │팀 C │  │팀 D │  ...                            │
│  └─────┘  └─────┘  └─────┘  └─────┘                                │
└────────────────────────────┬────────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│  3단계: 충돌 분석                                                     │
│  - 일치점 / 상충점 / 보완점 식별                                       │
└────────────────────────────┬────────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│  4단계: 크로스 리뷰 (병렬)                                             │
│  팀 A → 팀 B 피드백                                                  │
│  팀 B → 팀 C 피드백                                                  │
│  팀 C → 팀 A 피드백                                                  │
└────────────────────────────┬────────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│  5단계: 합의 도출                                                     │
│  - 합의/미합의 사항, 후속 조치                                         │
└────────────────────────────┬────────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────────┐
│  6단계: 결과 문서화                                                   │
│  docs/discussions/<의제>/                                            │
└─────────────────────────────────────────────────────────────────────┘
```

## 트리거 키워드

토론, 논의, 협업, 합의, 회의, 의사결정, discussion, 팀 미팅

## 참여 가능 팀

### 경영진 (Executive)
| 역할 | 전문 분야 |
|------|----------|
| CEO | 전략, 비전, 의사결정 |
| CTO | 기술 전략, 아키텍처 |
| CFO | 예산, 재무, ROI |
| COO | 운영 효율, 프로세스 |
| CPO | 제품 전략, 로드맵 |
| CMO | 마케팅, GTM |
| CISO | 보안 정책, 리스크 |
| CDO | 데이터 전략, AI/ML |
| CIO | IT 시스템, 디지털 전환 |
| CHRO | 인재, 조직 문화 |

## 결정 유형별 추천 팀

| 유형 | 자동 포함 팀 |
|------|-------------|
| 기술 | CTO, Backend, Frontend, DevOps, Security |
| 제품 | CPO, PM, UX-UI, Design |
| 운영 | COO, DevOps, SRE, Infra |
| 보안 | CISO, Security, Backend, DevOps |
| 데이터 | CDO, Data/AI, DBA, Backend |
| 긴급 | CEO, CTO, COO + 관련 팀 |

## 출력물

1. **README.md**: 토론 개요 및 결정 사항
2. **minutes.md**: 상세 회의록 (의견, 리뷰, 분석)
3. **action-items.md**: 후속 조치 목록

## 실행 방법

```
/team-discussion
```

또는 의제와 함께:
```
/team-discussion 마이크로서비스 전환 검토
```

## 핵심 기능

- ✅ **다중 팀 병렬 의견 수집**
- ✅ **충돌 자동 감지**
- ✅ **크로스 리뷰 (팀 간 피드백)**
- ✅ **합의 도출**
- ✅ **결과 문서화**
