# New Feature Orchestrator

## 역할

새로운 기능 개발 시 모든 팀이 협력하여 계획을 수립하는 오케스트레이터입니다.

## 워크플로우

```
┌─────────────────────────────────────────────────────────────────┐
│                    /new-feature 실행                             │
└─────────────────────────────┬───────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│               1. 기능 요구사항 수집 (AskUserQuestion)             │
│                  - 기능명, 설명, 우선순위                         │
└─────────────────────────────┬───────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│               2. 코드베이스 탐색 (Explore Agent)                  │
│                  - 기존 패턴 분석                                 │
│                  - 관련 모듈 파악                                 │
└─────────────────────────────┬───────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│               3. 팀별 병렬 논의 (Task - 4개 동시)                 │
├─────────────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐  ┌─────────┐ │
│  │  Backend    │  │   DevOps    │  │     QA      │  │Security │ │
│  │  Architect  │  │    Infra    │  │   Tester    │  │ Auditor │ │
│  │             │  │             │  │             │  │         │ │
│  │ - 설계      │  │ - 환경      │  │ - 테스트    │  │ - 보안  │ │
│  │ - 구조      │  │ - 배포      │  │ - 검증      │  │ - 취약점│ │
│  └─────────────┘  └─────────────┘  └─────────────┘  └─────────┘ │
└─────────────────────────────┬───────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│               4. 종합 계획 수립 (Plan Agent)                      │
│                  - 팀별 결과 통합                                 │
│                  - 우선순위 정리                                  │
│                  - 일정 수립                                      │
└─────────────────────────────┬───────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│               5. 문서 생성                                       │
│                  - docs/features/<기능명>/README.md              │
│                  - 작업 계획서                                    │
└─────────────────────────────────────────────────────────────────┘
```

## 실행 방법

```
/new-feature
```

또는 기능명과 함께:
```
/new-feature 사용자 알림 기능
```

## 팀별 역할

### Backend Team
- **Architect**: 시스템 설계, DB 스키마, API 설계
- **Developer**: 구현 계획, 기술적 고려사항
- **Reviewer**: 코드 품질 기준, 리뷰 체크리스트

### DevOps Team
- **Infra**: 인프라 요구사항, 환경 변수
- **Deployer**: 배포 계획, 마이그레이션
- **Monitor**: 모니터링 항목, 알림 설정

### QA Team
- **Tester**: 테스트 계획, 테스트 케이스
- **Analyst**: 성능 기준, 부하 테스트 계획

### Security Team
- **Auditor**: 보안 요구사항, OWASP 체크리스트
- **Pentester**: 보안 테스트 시나리오

## 출력물

1. **기능 문서**: `docs/features/<기능명>/README.md`
2. **API 문서**: `docs/features/<기능명>/api-reference.md`
3. **구현 계획**: `docs/features/<기능명>/implementation-plan.md`
4. **Todo 목록**: 작업 항목 목록

## 예시

### 입력
```
/new-feature 파일 업로드 기능
```

### 출력 구조
```
docs/features/file-upload/
├── README.md              # 기능 개요
├── api-reference.md       # API 엔드포인트
└── implementation-plan.md # 구현 계획
    - Backend 계획
    - DevOps 계획
    - QA 계획
    - Security 계획
```
